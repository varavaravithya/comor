พื้นฐานของบัส

บัสประกอบไปด้วยกลุ่มของสายควบคุม (Control lines) ใช้สำหรับสัญญาณ request และ acknowledgment และใช้เป็นตัวบอกประเภทของข้อมูลบนสายข้อมูล และกลุ่มของสายข้อมูล (Data lines) ใช้รับส่งข้อมูลระหว่างต้นทางถึงปลายทาง ซึ่งรวมถึงคำสั่งที่ซับซ้อน และแอดแดรสต่างๆ

บัสบางตัวจะแบ่งกลุ่มสายสัญญาณเป็นสองกลุ่ม เพื่อให้ส่งได้ทั้งข้อมูลและแอดเดรสภายในครั้งเดียวกัน ในกรณีนี้สายควบคุมจะถูกใช้ในการบอกว่าอะไรที่กำลังวิ่งอยู่ในบัส และทำหน้าที่เรียกใช้บัสโปรโตคอล และเพราะว่าบัสเป็นระบบที่มีการใช้งานร่วมกันกัน เราจึงต้องมีโปรโตคอลที่เหมาะสมในการตัดสินใจว่าใครจะใช้เป็นคนถัดไป

การทำงานหลักๆ ของบัสประกอบด้วยสองขั้นตอนคือการส่งแอดเดรสและการรับหรือส่งข้อมูล นิยามจากการทำงานที่กระทำกับหน่วยความจำ คือ ขั้นตอนการอ่านจะทำการส่งข้อมูลจากหน่วยความจำไปยังหน่วยประมวลผลและอุปกรณ์ I/O อื่นๆ และขั้นตอนการเขียน จะทำการเขียนข้อมูลลงบนหน่วยความจำ เพื่อหลีกเลี่ยงความสับสนเราจะเรียกว่า output และ input ตามลำดับ ซึ่งเป็นการอ้างอิงจากการทำงานของหน่วยประมวลผล กล่าวคือ input จะหมายถึงกระบวนการส่งข้อมูลจากอุปกรณ์ I/O ต่างๆ ไปยังหน่วยความจำที่ซึ่งหน่วยประมวลผลสามารถอ่านได้ และ output จะหมายถึงกระบวนการส่งข้อมูลไปยังอุปกรณ์ต่างๆ จากหน่วยความจำที่ซึ่งหน่วยประมวลผลสามารถเขียนได้

บัสแบ่งประเภทออกเป็น Processor-memory Buses กับ I/O buses โดย Processor-Memory buses จะมีขนาดสั้นและมีความเร็วสูง ในขณะที่ I/O buses จะมีความยาวมากกว่า และ สามารถเชื่อมต่ออุปกรณ์ได้หลายประเภทซึ่งอาจมีขนาดของ bandwidth ในการส่งข้อมูลที่แตกต่างกัน บัส I/O จะไม่สามารถติดต่อหน่วยความจำได้โดยตรงแต่จะใช้ Processor-Memory หรือ Backplane Bus ในการติดต่อกับหน่วยความจำแทน สำหรับบัสแบบอื่นๆ ที่มีลักษณะแตกต่างไปจากนี้จะถูกจัดไว้ในกลุ่มพิเศษ เช่น graphic bus

เนื่องจากบัส I/O ใช้เพื่อรองรับการขยายตัวของอุปกรณ์และสนับสนุนอุปกรณ์ชนิดใหม่ ทำให้ต้องมีการกำหนดเป็นมาตราฐาน ใช้บ่งบอกคุณลักษณะของบัส เพื่อประโยชน์ต่อการที่นักพัฒนาจะออกแบบอุปกรณ์ชนิดใหม่ขึ้นมา และสามารถรับประกันได้ว่าสามารถเชื่อมต่อเข้ากับระบบที่มีอยู่ได้อย่างดี

ระบบพื้นฐานสำหรับการติดต่อสื่อสารบนบัสมีสองประเภท ได้แก่ Synchronous และ Asynchronous ถ้าเป็นแบบ Synchronous จะรวมสัญญาณนาฬิกาเข้าไปในสายควบคุมและบังคับโปรโตคอลสำหรับการติดต่อสื่อสารให้สัมพันธ์กับสัญญาณนาฬิกา การติดต่อลักษณะนี้มีข้อเสียหลักๆ อยู่สองอย่าง อย่างแรก อุปกรณ์ทุกตัวบนบัสจำเป็นต้องทำงานที่อัตราสัญญาณนาฬิกาเดียวกัน สอง เนื่องจากปัญหา Clock Skew ทำให้บัสลักษณะนี้มีความยาวมากไม่ได้ที่ความเร็วสูง Processor-memory ส่วนใหญ่เป็นแบบ Synchronous เพราะอุปกรณ์ที่ติดต่อกันอยู่ใกล้กันมาก, มีจำนวนน้อย และถูกออกแบบมาให้ทำงานที่อัตราสัญญาณนาฬิกาสูง

Asynchronous Bus ไม่ใช้สัญญาณนาฬิกา ทำให้สามารถเชื่อมต่ออุปกรณ์ได้หลากหลาย มีความยาวได้มากกว่า โดยไม่ต้องกังวลในเรื่องของปัญหา Clock Skew และ Synchronization ทั้ง Firewire และ USB 2.0 เป็นแบบ Asynchronous ในการส่งข้อมูลระหว่างกันจะใช้ Handshaking Protocol ประกอบด้วยลำดับของขั้นตอนซึ่งทั้งผู้ส่งและผู้รับจะดำเนินการขั้นต่อไปได้ก็ต่อเมื่อทั้งสองฝ่ายยอมรับ รูป 8.3 แสดงการทำงานของโปรโตคอลโดยอธิบายจากขั้นตอนในการติดต่อสื่อสาร

แม้ว่า Bandwidth ของบัสจะขึ้นอยู่กับการทำงานของโปรโตคอลว่าเป็นแบบ Synchronous หรือ Asynchronous และ Timing Characteristic ของบัสแล้ว ยังมีปัจจัยอื่นๆ ที่มีผลกระทบต่อ Bandwidth อีกหลายตัว ที่สำคัญคือความกว้างของ Data Bus และขึ้นอยู่กับว่าแต่ละครั้งของการส่งข้อมูลสนับสนุนการส่งข้อมูลแบบ Block หรือ Word

วิธีการอื่นในการเพิ่มประสิทธิภาพ Bandwidth บนบัสคือการปล่อยบัสให้ว่างในขณะที่ไม่มีการรับส่งข้อมูล โปรโตคอลลักษณะนี้เรียกว่า Split Transaction Protocol ข้อดีคือขณะที่ไม่มีการส่งข้อมูล บัสจะว่าง และโปรโตคอลจะอนุญาตให้คนอื่นสามารถใช้งานได้ ซึ่งจะช่วยเพิ่มประสิทธิภาพให้กับ Bandwidth ในระบบที่หน่วยความจำต้องทำงานหลายๆ งานที่มีการทับซ้อนกัน
