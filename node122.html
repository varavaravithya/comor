Data Hazard

Data Hazard เกิดในกรณีที่ไพพ์ลายน์ต้องหยุดทำงานเพื่อให้คำสั่งก่อนหน้าทำงานเสร็จก่อนที่จะดำเนินการประมวลผลคำสั่งปัจจุบัน ในโปรเซสเซอร์ การเกิด Data Hazard เกิดจากการที่คำสั่งที่ตามมาต้องใช้ผลลัพธ์ที่เกิดขึ้นจากการประมวลผลของคำสั่งก่อนหน้าที่ยังอยู่ในไพพ์ลายน์ (Data Dependency) ตัวอย่างเช่น 

   add  $s0, $t0, $t1
   sub  $t2, $s0, $t3

ถ้าไม่มีการจัดการอย่างหนึ่งอย่างใด การเกิด Data Hazard จะส่งผลต่อประสิทธิภาพของโปรเซสเซอร์เป็นอย่างมาก เนื่องจาก Data Dependency เกิดขึ้นมากในโปรแกรมทั่วไป อย่างไรก็ตามในหลายกรณี เราไม่จำเป็นต้องรอให้คำสั่งก่อนหน้าทำงานจนเสร็จสิ้นก่อนที่จะประมวลผลคำสั่งปัจจุบัน โดยการป้อนกลับค่าที่พึ่งประมวลผลเาร็จกลับเข้ามาให้คำสั่งก่อนหน้า ซึ่งวิธีการนี้เรียกว่า การทำ Forwarding หรือ Bypassing
