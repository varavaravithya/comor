เปรียบเทียบประสิทธิภาพการทำงานของการควบคุมแบบต่างๆ

ทำการเปรียบเทียบประสิทธิภาพของการควบคุม Single-Cycle, Multicycle และ แบบ Pipeline โดยใช้ค่าเฉลี่ยของชุดคำสั่งที่ได้มาจากโปรแกรม SPECTINT2000 สมมุติให้ครึ่งหนึ่งของคำสั่ง load มีคำสั่งที่ตามมาใช้ผลจากการ load นั้น และการเกิด Branch Delay 1 วงรอบการทำงาน และหนึ่งในสี่ของการทำ Branch มีการคาดเดาผิด สมมุติให้คำสั่ง jump มีการ Delay หนึ่งวงรอบ ดังนั้นมีใช้เวลาเฉลี่ยสองวงรอบการทำงาน และไม่คำนึงถึง Hazard อื่นๆ

ให้ส่วนประกอบแต่ละส่วนใช้เวลาทำงานดังต่อไปนี้

  • 200 ps สำหรับ Memory Access
  • 100 ps สำหรับ ALU Operation
  • 50 ps สำหรับ Register File Read/Write

สำหรับ Single-Cycle จะมีวงรอบการทำงานเท่ากับ


(6.3)

และแต่ละประเถทคำสั่งมีความถี่ดังต่อไปนี้

  • 20% loads
  • 10% stores
  • 11% branches
  • 2% jumps
  • 52

วงรอบการทำงานของโปรเซสเซอร์แบบ Multicycle และ Pipeline เท่ากับ

สำหรับโปรเซสเซอร์ Pipeline คำสั่ง load ใช้เวลา 1 วงรอบ ในกรณีที่ไม่มี Dependence และ 2 วงรอบในกรณีที่มี Dependence จึงได้ค่าเฉลี่ยเท่ากับ 2 สำหรับคำสั่ง Branch มีค่าเฉลี่ยเท่ากับ 1.25 วงรอบการทำงาน ALU ใช้ 1 วงรอบการทำงาน และ jump ใช้สองวงรอบการทำงาน จะสามารถคำนวณ CPI ได้เท่ากับ


(6.4)

โดยเวลาเฉลี่ยในการประมวลผลของโปรเซสเซอร์ไพพ์ลายน์เท่ากับ ในขณะที่แบบ Single Cycle เท่ากับ และ Multicycle เท่ากับ
