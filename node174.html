กล่าวนำ

สำหรับอุปกรณ์ที่เกี่ยวข้องกับการเก็บข้อมูลนั้นหากเกิดเหตุการณ์ไม่คาดคิดจนทำให้ข้อมูลที่เก็บอยู่เกิดการสูญหายขึ้นแล้ว อาจเป็นเรื่องที่ทำให้ผุ้ใช้งานต้องสูญเสียอย่างมาก ดังนั้นความสามารถในการพึ่งพาได้ (Dependability) ของอุปกรณ์เก็บข้อมูลต้องมีมากกว่าในส่วนของการคำนวณ จึงเป็นเหตุการเลือกใช้อุปกรณ์ที่เกี่ยวกับ I/O เน้นไปทางความน่าไว้วางใจและราคา ส่วนหน่วยประมวลผลและหน่วยความจำจะเน้นไปทางประสิทธิภาพและราคา

ระบบของ I/O จะต้องมีการวางแผน เพื่อให้สามารถขยาย และรองรับอุปกรณ์ที่หลากหลาย ซึ่งไม่มีส่วนเกี่ยวข้องกับส่วนประมวลผล ความสามารถในการขยายนี้ รวมถึงความจุของการเก็บข้อมูล ที่เป็นอีกปัจจัยสำหรับระบบ I/O ด้วย แม้ว่าระบบอาจจะไม่ค่อยได้ติดต่อกับหน่วยเก็บข้อมูล (Disk Storage) ก็ตาม อย่างไรก็ดีระบบ I/O ก็ยังมีผลกระทบต่อประสิทธิภาพโดยตรง ตัวอย่างเช่น อุปกรณ์บางประเภท คำนึงถึงความสามารถในการเข้าถึง (Access Latency) ในขณะที่บางประเภทคำนึงถึงความสำคัญของอัตราความสำเร็จ (Throughput) เป็นหลัก นอกจากนี้ประสิทธิภาพยังเกี่ยวด้านอื่นของระบบอื่นอีกเช่น ลักษณะเฉพาะของอุปกรณ์ (Device Characteristic), จุดเชื่อมต่อระหว่างอุปกรณ์กับระบบอื่นๆ, การจัดลำดับขั้นของหน่วยความจำ (Memory Hierarchy) รวมทั้งประเภทของระบบปฏิบัติการด้วย ในรูปที่ 8.1 แสดงถึงโครงสร้างพื้นฐานของระบบ I/O ทุกส่วนประกอบตั้งแต่อุปกรณ์ I/O ที่ต่อกับหน่วยประมวลผลไปจนถึงการจัดการระบบจะส่งผลกระทบต่อความน่าไว้วางใจ ความสามารถในการต่อขยาย และประสิทธิภาพในการทำงานของส่วนที่มีอุปกรณ์ I/O ด้วย

Figure 8.1: โครงสร้างพื้นฐานของระบบ I/O

อุปกรณ์ I/O มีมากมายหลายประเภท ซึ่งมีคุณลักษณะที่ใช้บ่งบอก 3 ประการดังนี้

  • ลักษณะการทำงาน ทำหน้าที่ในการนำข้อมูลเข้า (input), ส่งข้อมูลออก (output) หรือเก็บข้อมูล (storage)
  • ผู้ดำเนินการ เป็นผู้ที่กระทำเกี่ยวกับ I/O ได้แก่มนุษย์ หรือเครื่องจักร
  • อัตราข้อมูล อัตราการส่งถ่ายข้อมูลสูงสุดระหว่างอุปกรณ์ I/O กับหน่วยความจำ

ตัวอย่างเช่น แป้นพิมพ์เป็นอุปกรณ์นำข้อมูลเข้าถูกใช้โดยมนุษย์มีอัตราส่งถ่ายข้อมูลสูงสุดประมาณ 10 ไบต์ต่อวินาที ในขณะที่การแสดงผลทางกราฟฟิกใช้การส่งข้อมูลเป็นอย่างมาก

สำหรับการประเมินประสิทธิภาพของอุปกรณ์ I/O นั้นโดยมากจะใช้โปรแกรมประยุกต์เป็นตัววัด ซึ่งในบางระบบจะคำนึงถึงปริมาณข้อมูลเป็นหลัก โดยในกรณีนี้ความกว้างของช่องสัญญาณจะสำคัญที่สุด โดยความกว้างของช่องสัญญาณสามารถวัดได้สองวิธีดังนี้

  1. วัดจากปริมาณข้อมูลที่ผ่านเข้าระบบในช่วงระยะเวลาหนึ่ง
  2. วัดจากจำนวนคำสั่งของ I/O ที่ทำได้ต่อหนึ่งหน่วยเวลา

ซึ่งวิธีการวัดที่ดีที่สุดนั้นขึ้นอยู่กับว่าอยู่ในสภาพแวดล้อมใด ยกตัวอย่างเช่นงานเกี่ยวกับมัลติมีเดียจะเป็นข้อมูลที่มีความยาวมากซึ่งต้องอาศัยการส่งถ่ายข้อมูลที่สูง แต่งานบางประเภทที่มีขนาดของข้อมูลไม่ใหญ่มากนักแต่มีจำนวนของข้อมูลจำนวนมากซึ่งต้องอาศัยความเร็วในการทำงานต่อคำสั่งที่รวดเร็ว

สำหรับแอพพลิเคชั่นอื่นอาจจะสนใจในเรื่องของการตอบสนองที่รวดเร็วเนื่องจากเป็นงานที่มีการสั่งงานแบบเล็กๆน้อยๆ ซึ่งในกรณีนี้การทำงานของอุปกรณ์ I/O จึงต้องอาศัยอุปกรณ์ที่มีความสามารถในการเข้าถึงข้อมูลอย่างรวดเร็วเพื่อให้ระบบสามารถทำงานได้อย่างมีประสิทธิภาพ
