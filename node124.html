การจัดเรียงคำสั่งใหม่เพื่อหลีกเลี่ยงการ Stall

พิจารณาส่วนของโปรแกรมภาษา C ดังต่อไปนี้ 

   A = B + E;
   C = B + F;

สามารถเขียนเป็นภาษาเอสเซมบลี่ของ MIPS ได้ดังนี้ (สมมุติให้มี offset อยู่ที่รีจีสเตอร์ $t0) 

  lw  $t1, 0($t0)
  lw  $t2, 4($t0)
  add $t3, $\t1, $t2
  sw  $t3, 12($t0)
  lw  $t4, 8($t0)
  add $t5, $t1, $t4
  sw  $t5, 16($t0)

จงจัดเรียงคำสั่งใหม่ให้ไม่เกิด Hazard และยังมีการทำงานถูกต้อง

คำสั่ง add ทั้งสองคำสั่งก่อให้เกิด Data Hazard และ Stall เนื่องจากต้องใช้ค่าจากคำสั่ง lw ที่มาก่อนหน้า เราสามารถจัดเรียงคำสั่งไม่ให้เกิด Hazard ได้โดยการแยกคำสั่ง lw ออกจากกัน ดังแสดงต่อไปนี้ 

  lw  $t1, 0($t0)
  lw  $t2, 4($t0)
  lw  $t4, 8($t0)
  add $t3, $\t1, $t2
  sw  $t3, 12($t0)
  add $t5, $t1, $t4
  sw  $t5, 16($t0)

ซึ่งจากชุดคำสั่งขั้นต้นจะทำงานเร็วกว่าสองวงรอบการทำงาน
