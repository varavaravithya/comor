การส่งคำสั่งไปยังอุปกรณ์ I/O

ในการส่งคำสั่งติดต่อไปยังอุปกรณ์ หน่วยประมวลผลต้องสามารถเข้าถึงแอดเดรสของอุปกรณ์และจัดเตรียมหนึ่งคำสั่งหรือมากกว่า วิธีการที่ใช้ในการอ้างถึงแอดเดรสของอุปกรณ์มี 2 วิธี คือ memory-mapped I/O และ special I/O instructions ใน memory-mapped I/O แอดเดรสส่วนที่ว่างจะถูกใช้โดยอุปกรณ์ I/O การอ่านและเขียนไปยังแอดเดรสนั้นจะถูกตีความเป็นคำสั่งส่งไปยังอุปกรณ์

ตัวอย่างเช่น กระบวนการเขียนสามารถใช้ส่งข้อมูลไปยังอุปกรณ์ที่ซึ่งข้อมูลจะถูกตีความในรูปของคำสั่ง เมื่อหน่วยประมวลผลวางแอดเดรสและข้อมูลบนบัสของหน่วยความจำ ระบบหน่วยความจำจะปฏิเสธการทำงานเพราะแอดเดรสที่บ่งชี้ไปยังส่วนของพื้นที่ว่างในหน่วยความจำถูกใช้สำหรับ I/O อย่างไรก็ตาม device controller จะบันทึกข้อมูลนั้น และส่งไปยังอุปกรณ์ในลักษณะของคำสั่ง โปรแกรมของผู้ใช้จะถูกหลีกเลี่ยงจากการทำงานกับ I/O โดยตรงเพราะระบบปฏิบัติการไม่ได้จัดเตรียมการเข้าถึงแอดเดรสส่วนที่ว่างที่ใช้โดยอุปกรณ์ และแอดเดรสก็ถูกป้องกันด้วยวิธีการแปลงแอดเดรส วิธีการ memory-mapped I/O จะถูกใช้ส่งข้อมูลโดยการเขียนหรืออ่านไปยังแอดเดรสที่เลือกไว้ อุปกรณ์ที่ใช้แอดเดรสนั้นจะหาว่าเป็นคำสั่งแบบไหน และข้อมูลอาจจะเตรียมไว้โดยการเขียนหรือได้รับจากการอ่าน ในหลายๆ กรณี แอดเดรสมีการเข้ารหัสไว้ทั้ง ID ของอุปกรณ์และประเภทของการส่งผ่านระหว่างหน่วยประมวลผลกับอุปกรณ์ I/O

ในการทำงานจริงสำหรับการอ่านและเขียนข้อมูล โปรแกรมจะมีการทำงานร่วมกับอุปกรณ์หลายตัว หน่วยประมวลผลอาจจะต้องมีตัวบอกสถานะของอุปกรณ์ โดยจะเป็นคำสั่งเฉพาะเพื่อหาว่าคำสั่งที่ส่งไปทำงานนั้นเสร็จสมบูรณ์หรือไม่ ตัวอย่างเช่นเครื่องพิมพ์ที่ต้องมี I/O register ขนาดหนึ่งบิท ที่ใช้เก็บสถานะของเครื่องพิมพ์เพื่อแจ้งต่อหน่วยประมวลผลว่าพิมพ์สำเร็จหรือเกิดความผิดพลาดขึ้น

อีกวิธีการหนึ่งในการทำ memory-mapped I/O คือใช้วิธีการส่ง I/O instructions ให้กับหน่วยประมวลผล ซึ่ง I/O instructions เหล่านี้สามารถระบุทั้งจำนวนอุปกรณ์และชุดคำสั่งหรือตำแหน่งของชุดคำสั่งในหน่วยความจำ ตัวอย่างคอมพิวเตอร์ที่ใช้ I/O instructions ได้แก่ Intel IA-32 และ IBM 370
