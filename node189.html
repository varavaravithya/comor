การเชื่อมต่ออุปกรณ์

บัสหรือเน็ตเวิร์กโปรโตคอลนิยามจากข้อมูลที่มีขนาดเป็น word หรือ block ว่าควรจะติดต่อสื่อสารกันบนสายได้อย่างไร สิ่งเหล่านี้ยังได้กล่าวอ้างไปถึงงานอื่นๆ ที่จำเป็นต้องมีการส่งข้อมูลจากอุปกรณ์และจากภายในแอดเดรสว่างในหน่วยความจำของบางโปรแกรมของผู้ใช้ ในบทนี้เราจะเน้นถึงงานเหล่านี้และการตอบคำถามที่ว่า

  • ผู้ใช้เปลี่ยนการร้องขอให้อยู่ในรูปคำสั่งสำหรับอุปกรณ์และทำการติดต่อสื่อสารกับอุปกรณ์ได้อย่างไร?
  • ข้อมูลถูกส่งไปหรือกลับจากตำแหน่งในหน่วยความจำได้อย่างไร?
  • อะไรคือบทบาทของระบบปฏิบัติการ?

จะเห็นว่าในการจะตอบคำถามเหล่านี้ ระบบปฏิบัติการจะเป็นตัวหลักในการทำงานร่วมกับอุปกรณ์ I/O ทำหน้าที่ในการประสานระหว่าง hardware กับโปรแกรมที่เรียกใช้อุปกรณ์ I/O

ภาระหน้าที่ของระบบปฏิบัติการได้จากคุณลักษณะของระบบ I/O 3 ประการ

  1. โปรแกรมหลายตัวใช้ระบบ I/O ร่วมกันโดยทำงานผ่านหน่วยประมวลผล
  2. ระบบ I/O บ่อยครั้งที่ใช้การ interrupt เพื่อส่งข้อมูลเกี่ยวกับการทำงานของ I/O เพราะว่า interrupt จะส่งไปยัง kernel หรือ supervisor mode ซึ่งต้องทำงานผ่านระบบปฏิบัติการ
  3. การควบคุมในระดับล่างกับอุปกรณ์มีความซับซ้อน เพราะว่าต้องจัดการกับ concurrent event ที่เกิดขึ้น และเพราะบ่อยครั้งที่การร้องขอสำหรับการควบคุมอุปกรณ์ที่ถูกต้องมีรายละเอียดมาก

จากคุณลักษณะของระบบ I/O ที่กล่าวถึงตามข้างต้นได้นำไปสู่ฟังก์ชั่นต่างๆ ที่ระบบปฏิบัติการจำเป็นต้องเตรียมไว้

  • ระบบปฏิบัติการต้องรับประกันว่าโปรแกรมของผู้ใช้เข้าถึงได้เฉพาะส่วนของอุปกรณ์ที่ซึ่งผู้ใช้มีสิทธิ์เท่านั้น เช่น ระบบปฏิบัติการต้องไม่อนุญาตให้โปรแกรมอ่านหรือเขียนไฟล์บนดิสก์ถ้าเจ้าของไฟล์ไม่ได้ใช้งานโปรแกรมนี้อยู่ ในระบบที่มีการใช้อุปกรณ์ร่วมกัน การป้องกันจะไม่สามารถกระทำได้ถ้าผู้ใช้โปรแกรมสามารถเข้าถึงอุปกรณ์ได้โดยตรง
  • ระบบปฏิบัติการจัดเตรียม concept สำหรับการเข้าถึงอุปกรณ์โดยจัดเตรียม routines ที่ทำงานกับอุปกรณ์ในระดับล่าง
  • ระบบปฏิบัติการเรียกใช้ interrupt ที่สร้างโดยอุปกรณ์ I/O เหมือนกับที่เรียกใช้ exception ที่สร้างโดยโปรแกรม
  • ระบบปฏิบัติเข้าใช้งานทรัพยากร I/O ร่วมกันอย่างยุติธรรมให้ดีเท่ากับตารางการเข้าถึงตามลำดับเพื่อเพิ่ม throughput ของระบบ

ในการปฏิบัติตามฟังก์ชั่นเหล่านี้ให้เป็นประโยชน์ต่อโปรแกรมของผู้ใช้ ระบบปฏิบัติการจำเป็นต้องมีการติดต่อกับอุปกรณ์ I/O และหลีกเลี่ยงให้โปรแกรมของผู้ใช้ทำการติดต่อกับอุปกรณ์โดยตรง การติดต่อสื่อสาร 3 ลักษณะที่จำเป็นคือ

  1. ระบบปฏิบัติการจำเป็นต้องส่งคำสั่งไปยังอุปกรณ์ ซึ่งคำสั่งเหล่านี้ไม่ได้รวมถึงเฉพาะแค่การทำงานเช่นอ่านและเขียน แต่รวมไปถึงการทำงานต่างๆ ที่เกิดขึ้นบนอุปกรณ์ เช่น การค้นหาดิสก์ (disk seek)
  2. อุปกรณ์จำเป็นต้องแจ้งระบบปฏิบัติการเมื่ออุปกรณ์ทำงานเสร็จสมบูรณ์หรือเกิดความผิดพลาดขึ้น
  3. ข้อมูลจำเป็นต้องถูกส่งระหว่างหน่วยความจำกับอุปกรณ์ I/O
