การวัดประสิทธิภาพ

เวลาในการประมวลผลเป็นตัววัดประสิทธิภาพของคอมพิวเตอร์ คอมพิวเตอร์ที่ทำงานที่กำหนดให้ในเวลาที่สั้นที่สุด ถือว่าเป็นคอมพิวเตอร์ที่เร็วที่สุด เวลาในการประมวลผลของโปรแกรม (Program Execution Time) วัดเป็น วินาทีต่อโปรแกรม การวัดเวลาสามารถแบ่งได้เป็น เวลานาฬิกา (Wall-Clock Time), เวลาตอบสนอง (Response Time), และเวลาที่ใช้ (Elapsed Time)

คอมพิวเตอร์ในปัจจุบันมักทำงานประมวลผลหลายโปรแกรมพร้อมกัน ในกรณีดังกล่าวเรามักจะทำการปรับปรุงอัตราความสำเร็จของงาน (Throughput) มากกว่าเวลาที่ใช้สำหรับโปรแกรมนั้นๆ เราสามารถแบ่งเวลาของโปรเซสเซอร์ในการคำนวณงานที่ทำการพิจารณาอยู่เรียกว่า CPU Execution Time หรือ CPU Time ซึ่งเป็นเวลาที่โปรเซสเซอร์ใช้ในการประมวลผลงานนั้นๆ และไม่รวมถึงเวลาที่รอการตอบสนองจาก I/O หรือ รอโปรแกรมอื่นๆ CPU Time สามารถแบ่งย่อยไปอีกเป็น User CPU Time และ System CPU Time โดยที่ System CPU Time เป็นเวลาที่ระบบปฏิบัติการทำงานเพื่อบริการโปรแกรมที่ทำงานวัดประสิทธิภาพ อย่างไรก็ตาม ในปัจจุบันเป็นการยากที่จะแยกระหว่าง User CPU Time และ System CPU Time เนื่องจากระบบปฏิบัติการมีความซับซ้อนสูงขึ้น

เพื่อความสอดคล้องในการพิจารณาประสิทธิภาพ เราพิจารณาประสิทธิภาพของระบบ (System Performance) ว่าเป็นเวลาที่ใช้ในการคำนวณบนระบบที่ไม่มีภาระงานอื่นนอกเหนือจาก โปรแกรมที่พิจารณา และ ประสิทธิภาพของตัวประมวลผล (CPU Performance) เป็น CPU Time

บางครั้งเราพิจารณาประสิทธิภาพเป็นเวลาที่คอมพิวเตอร์ที่ทำงานพื้นฐานเสร็จ ต่อวงรอบการทำงานหรือเรียกว่า Clock Cycle โดยมีระยะเวลาระหว่างวงรอบเรียกว่า Clock Period
